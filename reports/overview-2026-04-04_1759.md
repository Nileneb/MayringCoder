### tests/Feature/Recherche/PhaseP4Test.php
## Zusammenfassung
Wie man bereits aus den Tests sehen kann, können wir diese Funktionalität sehr schnell mit Livewire implementieren. In unserem Fall ist es nicht notwendig, eine Tabelle oder ein Model zu erstellen, da wir das Ganze nur per Formular befüllen und anschließend abgesendet überprüfen. Durch dieses Vorgehen können wir diese Funktionalität komplett aus dem Routing heraus lösen, da Livewire dann uns eine einfache Art von API bereitstellt.

Wenn wir das Ganze so implementieren würden, wären wir jedoch nicht mehr in der Lage, den Zugriff dieser Funktionalität durch die URL zu steuern oder über ein einfaches Klickmenü auszustatten. Da Livewire für die Anfrage des Requests noch nie aufgerufen hat, kann es sein, dass diese Funktionsweise zuvor noch nicht erkannt wird. Um das zu vermeiden, müssen wir die Funktionalität der Phase in einen eigenen Controller umstellen und steuern können:

```php
// App\Http\Controllers\P4SuchstringsController.php
<?php namespace App\Http\Controllers;

use App\Models\User;
use App\Models\Projekt;
use App\Models\P4Suchstring;
use Illuminate\Support\Facades\Auth;
use Livewire\WithPagination;

class P4SuchstringsController extends Controller {
    use WithPagination;

    // ─── Authorization ─────────────────────────────────────────
    /**
     * Wir prüfen, ob die aktuelle Anmeldung mit dem User übereinstimmt, der das Projekt angehört.
     */
    public function authorize() {
        $user = Auth::id();

        if ($user === null) {
            return redirect()->to('/login');
        }

        abort_if(Projekt::where('id', $this->projektId)->where('user_id', $user)->first() == false, 403);
    }

    /**
     * Die Route /phase-p4 ist nicht für alle aufgerufen.
     */
    public function forbiddenResponse() {
        return redirect()->to('/forbidden');
    }

    // ─── Datenmodell ─────────────────────────────────────────
    /**
     * Alle Suchstrings einer Phase zurückgeben.
     *
     * @return Illuminate\Database\Eloquent\Collection<App\Models\P4Suchstring>
     */
    public function getAll() {
        $this->authorize();

        return P4Suchstring::where('projekt_id', $this->projektId)->get();
    }

    // ─── Ansichten ────────────────────────────────────────────
    /**
     * Die Ansicht für das Erstellen einer Suchstring.
     *
     * @return View
     */
    public function create() {
        return view('recherche/phase-p4');
    }
}
```

Wir erstellen in diesem Controller zwei Methoden: **authorize** und **forbiddenResponse**. Die erste Prüft auf die Identität des Users, der das Projekt angehört. Ist dieser nicht derselbe wie der User der eingeloggt ist, so wird er weitergeleitet zu **/forbidden**, ansonsten werden die Suchstrings zurückgegeben und die Ansicht für das Erstellen einer Suchstring geladen:

Wir erstellen in diesem Controller nur eine Methoden: **missingRouteResponse**. Wenn ein Request für einen fremden Controller aufgerufen wird, erscheint die 404 Seite. Nachdem wir jetzt alle Ansichten mit Livewire übersetzen können, können wir den Zugriff aller Routes einschränken und eine Funktionalität nur noch ausführen, wenn sie aus dem Routing heraus aufgerufen wird:

```php
<?php namespace App\Http\Middleware;

use Closure;
use Illuminate\Support\Facades\Auth;
use App\Models\Projekt;
use Livewire\Livewire;

class P4Authorization {
    /**
     * Wir prüfen, ob die aktuelle Anmeldung mit dem User übereinstimmt, der das Projekt angehört.
     */
    public function handle($request, Closure $next) {
        $user = Auth::id();

        if ($user === null) {
            return redirect()->to('/login');
        } else if (Projekt::where('id', 4)->where('user_id', $user)->first() == false) {
            return redirect()->to('/forbidden');
        }

        // Der Request kann ausgeführt werden. Nun schauen wir mal, was es für eine Route gibt:
        $routeName = $request->segment(1);

        if ($routeName === 'phase-p4') {
            // Unsere Route ist /phase-p4
            return Livewire::component('recherche/phase-p4', ['projekt' => 4]);
        }

        return $next($request);
    }
}
```

```php
<?php namespace App\Providers;

use App\Http\Middleware\P4Authorization;
use Illuminate\Auth\Notifications\VerifyEmail;
use Illuminate\Foundation\Support\Providers\RouteServiceProvider as ServiceProvider;
use Illuminate\Support\Facades\Blade;
use Livewire\Livewire;

class RouteServiceProvider extends ServiceProvider {
    /**
     * Wir setzen unseren P4Authorization-Middleware, der uns das Verhindern von einfachen Zugriffsmöglichkeiten für die einzelnen Routen erlaubt.
     */
    public function boot() {
        $this->configureRateLimiting();

        // Unsere Middleware setzen wir uns nochmal explizit
        $this->app['router']->aliasMiddleware('p4_authorization', P4Authorization::class);

        parent::boot();
    }

    /**
     * Der Methodenname wird beim Aufruf automatisch ergänzt: **verifyEmail**
     */
    public function register() {
        VerifyEmail::toMailUsing(function ($notifiable) {
            return new EmailVerificationNotification($notifiable->verificationUrl(URL::temporarySignedRoute('email.verify', now()->addMinutes(60), ['id' => $notifiable->getKey()])));
        });

        parent::register();
    }

    /**
     * Wir setzen unseren P4Authorization-Middleware, der uns das Verhindern von einfachen Zugriffsmöglichkeiten für die einzelnen Routen erlaubt.
     */
    public function map() {
        $this->mapApiRoutes();

        $this->mapWebRoutes();

        // Route für den User-Controller
        $this->app['router']->get('/user/logout', 'UserController@logout')->name('logout');

        /**
         * Alle Routen, die mit /phase-p4 beginnen, müssen vom P4Authorization-Middleware geprüft werden.
         */
        $this->app['router']->middleware('p4_authorization')->group(function ($router) {
            // Route für die Suchstrings
            $router->get('/phase-p4', 'P4SuchstringsController@create');
            $router->post('/phase-p4/newSs', 'P4SuchstringsController@create');
        });

        /**
         * Alle Routen, die mit /phase-p4-search beginnen, dürfen nur von Livewire aufgerufen werden.
         */
        $this->app['router']->middleware('livewire')->group(function ($router) {
            // Route für den Suchstrings
            $router->get('/phase-p4-search/{id}', 'P4SuchstringsController@show');
            $router->post('/phase-p4/newSs', 'P4SuchstringsController@create');
        });
    }

    /**
     * Unser Router bekommt eine Route für den Livewire-Request /phase-p4-search.
     */
    protected function mapWebRoutes() {
        // Route für die Suchstrings
        $this->app['router']->middleware('web')->group(function ($router) {
            $router->get('/phase-p4-search/{id}', 'P4SuchstringsController@show');
        });
    }

    /**
     * Alle Routen, die mit /phase-p4 beginnen, müssen vom P4Authorization-Middleware geprüft werden.
     */
    protected function mapApiRoutes() {
        // Route für die Suchstrings
        $this->app['router']->middleware('api')->group(function ($router) {
            $router->get('/phase-p4', 'P4SuchstringsController@index');
        });
    }
}
```

Hier wird unser Middleware **p4_authorization** eingeführt, der die Routen zusammenfasst, für die wir Livewire aufrufen können. Sobald die Anfrage gesendet wurde, verhält sich Livewire wie jede andere Anfrage und gibt die Daten des Models zurück. Da wir diese Route nur von Livewire aus aufrufen können, muss sie nicht in der config/livewire.php eingeschaltet sein.

```php
<?php
namespace App\Exceptions;
use Exception;
class Handler extends \App\Traits\HasException {

    /**
     * @param  \Illuminate\Http\Request  $request
     * @param  \Exception $exception
     * @return \Illuminate\Http\Response
     */
    public function render($request, Exception $exception) {
        if ($this->isHttpException($exception)) {
            return $this->renderHttpException($exception);
        }
        elseif ($this->isExceptionHandler($exception)) {
            return response()->view('errors.500', [], 500);
        }
        else {
            return parent::render($request, $exception);
        }
    }

    /**
     * @param  \Exception $e
     * @return bool
     */
    protected function isExceptionHandler(Exception $e) : bool {
        return ($this->isCoreExceptionHandler($e)) ||
               ($this->isRenderableExceptionHandler($e));
    }

    /**
     * @param  Exception $exception
     * @return string|null
     */
    public function convertExceptionToResponse(\Exception $exception): ?string {

        // If the exception is an App\Exceptions\BaseException, return its data
        if ($exception instanceof BaseException) {
            return json_encode($exception->data);
        }

        // Check if the exception is a HttpException with a code of 419
        elseif (($e = $this->getHttpExceptionFrom($exception)) && ($e->getStatusCode() == 419)) {

            // If the response is an App\Http\Response\AppResponse, return its data
            if (property_exists($response = $e->getResponse(), 'data')) {
                return json_encode($response->getData());
            }
        }

        // Else return the parent Exception message
        return parent::convertExceptionToResponse($exception);
    }
}
```
